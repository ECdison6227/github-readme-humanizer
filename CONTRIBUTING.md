# Contributing to github-readme-humanizer

Thanks for helping make READMEs more human.

## How to Contribute

1. Open an [Issue](https://github.com/ECdison6227/github-readme-humanizer/issues) to describe the bug or idea first.
2. Fork the repo and create a branch.
3. Make your changes.
4. Run a quick self-check:
   - Does the change follow the 12-point checklist in `SKILL.md`?
   - Are examples realistic but not inventing personal details about the maintainer?
   - Is the English README rewritten, not translated word-for-word?
5. Open a Pull Request with a clear description.

## Style Guide

- Avoid forbidden words: "powerful", "efficient", "intelligent", "elegant", "lightweight", "one-stop", "revolutionary".
- **Never invent the maintainer's backstory.** Ask the user for their real pain point, or use a clearly marked placeholder.
- Use real stories (user-provided), specific numbers, and concrete commands.
- Keep banners as draw.io-style SVG flowcharts.
- Contact email must remain `2014184720@qq.com` unless the maintainer explicitly changes it.
